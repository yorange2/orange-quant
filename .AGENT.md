# Orange Quant Agent Guide

## Project Overview

Orange Quant is an AI-powered quantitative trading framework built on Microsoft qlib. It supports:
- **Backtesting**: A-share (CSI300/CSI500) and crypto (Hyperliquid perpetuals, Binance spot)
- **Auto-trading**: Dockerized server with Binance mainnet execution
- **ML models**: LightGBM trainer + 15+ PyTorch models (LSTM, GRU, Transformer, etc.)
- **Data pipeline**: Hyperliquid API, Binance API, qlib data format

## Key Architecture

```
orange_quant/
├── data/         # Data download + Alpha factor handlers (crypto_download, handler)
├── models/       # LightGBM trainer + DLTrainer (LSTM, GRU, Transformer, etc.)
├── strategies/   # MomentumTopKStrategy (inherits qlib TopkDropoutStrategy)
├── backtest/     # BacktestRunner with configurable exchange costs
├── trading/      # BinanceBroker (paper/real) + StrategyRunner (auto-rebalance)
└── workflow/     # QuantExperiment + run_from_yaml() for YAML-driven experiments
```

## Commands

### Local Development
```bash
source .venv/bin/activate
pip install -e /path/to/qlib lightgbm pandas numpy pyyaml ccxt python-dotenv
```

### Experiments
```bash
# A-share CSI300
python -c "from orange_quant.workflow.experiment import run_from_yaml; run_from_yaml('config/workflow_config.yaml')"

# Crypto 187 perps (Hyperliquid)
python scripts/build_crypto_data.py
python -c "from orange_quant.workflow.experiment import run_from_yaml; run_from_yaml('config/workflow_config_crypto.yaml')"

# Crypto 9 U-spot
python -c "from orange_quant.workflow.experiment import run_from_yaml; run_from_yaml('config/workflow_config_crypto_spot.yaml')"

# Binance 20 blue-chips
python scripts/build_binance_data.py --top 50
python -c "from orange_quant.workflow.experiment import run_from_yaml; run_from_yaml('config/workflow_config_binance.yaml')"
```

### Auto Trading
```bash
# Docker
docker build -t orange-quant:latest .
docker run --rm --env-file .env orange-quant:latest --once --dry-run  # test
docker run --rm --env-file .env orange-quant:latest --once            # single rebalance
docker compose up -d                                                   # long-running server

# Local
python scripts/run_binance_testnet.py --dry-run
python scripts/server_entrypoint.py --once
```

## Data Locations

- `data/crypto_raw/` — Hyperliquid daily CSV (gitignored)
- `data/crypto_raw_1h/` — Binance hourly CSV (gitignored)
- `data/binance_raw/` — Binance daily CSV (gitignored)
- `data/qlib_data/crypto/` — qlib binary for Hyperliquid (gitignored)
- `data/qlib_data/crypto_1h/` — qlib binary for hourly (gitignored)
- `data/qlib_data/binance/` — qlib binary for Binance (gitignored)
- `~/.qlib/qlib_data/cn_data/` — A-share data (qlib official)

## Key Findings

1. **Quality > quantity**: 9 blue-chip coins (IC 0.15) >> 187 random coins (IC 0.04)
2. **Alpha158 is daily-only**: Hourly data with Alpha158 features gives IC ≈ 0
3. **Costs matter**: Binance 0.1% taker fee eats ~50% of excess returns
4. **Cross-sectional features need enough instruments**: Alpha158 rank features need 20+ coins
5. **Perp data for training, spot costs for backtest**: Valid for U-wrapped tokens on Hyperliquid

## Environment

- Python: 3.9+ with venv in `.venv/`
- macOS: requires `brew install libomp` for LightGBM
- Docker: `docker build -t orange-quant:latest .`
- API keys in `.env` (gitignored): `BINANCE_API_KEY`, `BIANCE_SECRET_KEY`

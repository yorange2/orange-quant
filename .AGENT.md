# Orange Quant Agent Guide

## Project Overview

Orange Quant is an AI-powered quantitative trading framework built on Microsoft qlib. It supports:
- **Backtesting**: A-share (CSI300) and Binance spot
- **Auto-trading**: Local and Dockerized server with Binance mainnet execution
- **ML models**: LightGBM + 15+ PyTorch models (LSTM, GRU, Transformer, etc.)
- **Data pipeline**: Binance API, qlib data format

## Key Architecture

```
orange_quant/
├── trading/      # BinanceBroker + StrategyRunner + ModelPredictor
└── workflow/     # QuantExperiment + run_from_yaml + run_dl_from_yaml
```

Thin-layer philosophy: model, strategy, and data handling are delegated to qlib directly via YAML config.

## Commands

### Setup
```bash
source .venv/bin/activate
pip install git+https://github.com/microsoft/qlib.git
pip install lightgbm pandas numpy pyyaml ccxt python-dotenv
# macOS: brew install libomp
```

### Experiments
```bash
# A-share CSI300
python scripts/csi300/build_data.py
python -c "from orange_quant.workflow.experiment import run_from_yaml; run_from_yaml('config/csi300-lgb-momtopk.yaml')"

# Binance spot
python scripts/biance/build_data.py --top 50
python -c "from orange_quant.workflow.experiment import run_from_yaml; run_from_yaml('config/binance-lgb-momtopk.yaml')"
```

### Auto Trading
```bash
# Local dry-run (recommended first)
python scripts/biance/execute.py --once --dry-run --model models/binance-lgb-momtopk.pkl

# Local live
python scripts/biance/execute.py --once --model models/binance-lgb-momtopk.pkl

# Docker
docker compose up -d
docker compose --profile dry-run up -d orange-quant-dry
```

## Data Locations

- `data/binance_raw/` — Binance daily CSV (gitignored)
- `data/qlib_data/binance/` — qlib binary for Binance (gitignored)
- `data/qlib_data/cn_data/` — A-share data (gitignored)

## Key Findings

1. **Costs matter**: Binance 0.1% taker fee eats ~50% of excess returns
2. **Cross-sectional features need enough instruments**: Alpha158 rank features need 20+ coins

## Environment

- Python: 3.9+ with venv in `.venv/`
- macOS: requires `brew install libomp` for LightGBM
- Docker: `docker compose up -d`
- API keys in `.env` (gitignored): `BINANCE_API_KEY`, `BIANCE_SECRET_KEY`
